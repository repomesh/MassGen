We're excited to release MassGen v0.1.87 — Documentation: Framework Comparisons & `llms.txt`! 🚀 Three new "MassGen vs ..." comparison pages (CrewAI, LangGraph, AutoGen/AG2), a curated `llms.txt` index plus full-corpus `llms-full.txt` dump for AI agents and crawlers (per [llmstxt.org](https://llmstxt.org) spec), and a one-line discriminator fix.

**Key Improvements:**

📚 **Framework Comparison Pages** — Three new "MassGen vs ..." pages under `docs/source/reference/comparisons/`:
- `crewai.rst`, `langgraph.rst`, `autogen.rst`
- Each page positions MassGen's parallel-refinement-with-voting model against the target framework's coordination shape, and lists when to reach for one versus the other
- The "How Does MassGen Compare?" section on the docs landing page now lists all four comparisons (LLM Council + the three new ones)
- Parent `comparisons.rst` drops "coming soon" and gains a toctree

🤖 **`llms.txt` for AI Agents** — Discoverability for crawlers:
- Curated [`llms.txt`](https://docs.massgen.ai/en/latest/llms.txt) index per [llmstxt.org spec](https://llmstxt.org), published at the docs site root via Sphinx `html_extra_path`
- Full-corpus [`llms-full.txt`](https://docs.massgen.ai/en/latest/llms-full.txt) dump (~1 MB across 59 files), generated by a Sphinx `build-finished` hook
- One-line pointer added to `README.md` for AI agents and crawlers

🔧 **`bootstrap_subagent` Single-Shot Fix** — `Orchestrator._run_bootstrap_discriminator_step` now passes `refine=False` to `spawn_subagent`. This is the canonical knob `SubagentManager` actually respects at the orchestrator level — without it, the orchestrator's `max_new_answers_per_agent: 3` default shadowed the coordination-dict overrides and let the discriminator refine instead of running single-shot.

**Getting Started:**

```bash
pip install massgen==0.1.87
```

- Read the comparisons: https://docs.massgen.ai/en/latest/reference/comparisons.html
- Point your AI agent at: https://docs.massgen.ai/en/latest/llms.txt
